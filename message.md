Мы выбрали тему "Web-приложение для составление конфигурации ПК.
В нём планирется реализовать следующий функционал:
1. Подбора комплектующих 
2. Возможность делиться сборками ПК с другими пользователями

Для ролевого поиска пока видится 3 вида пользователей:
1. Админ - сможет добавлять комплектующие в список доступных всем, удалять чужие посты и т.д.
2. Зарегистрированный  пользователь - может редактировать свои данные, просматривать чужие и комментировать чужие публикации 
3. Незарегистрированный пользователь - может просматривать чужие публикации
4. (Возможно) Заблокированный пользователь - может редактировать свои данные и просматривать чужие

В работе планируем использовать 3 класса СУБД:
1. PostgreSQL - для хранение основных данных 
Таблицы: users, builds, components, build_components, cpus, motherboards, gpus, rams, storages, power_supplies, cases, mice, keyboards, headphones, microphones, monitors, wishlists, wishlist_items. (18 таблиц).
2. MongoDB - для хранение информации о постах и комментариях к ним (т.к. комментарии могут иметь большую вложенность)
Таблицы: posts, comments (2 таблицы).
3. Redis - для кеширования частой информации о пользователях
Таблицы: user_sessions, recent_views (2 таблицы).

Итого суммарное кол-во таблиц пока равно 22.
